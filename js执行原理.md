# js执行原理

#### 老师解释的 同步和异步

```
同步：阻塞的
-张三叫李四去吃饭，李四一直忙得不停，张三一直等着，直到李四忙完两个人一块去吃饭
=浏览器向服务器请求数据，服务器比较忙，浏览器一直等着（页面白屏），直到服务器返回数据，浏览器才能显示页面
异步：非阻塞的
-张三叫李四去吃饭，李四在忙，张三说了一声然后自己就去吃饭了，李四忙完后自己去吃
=浏览器向服务器请求数据，服务器比较忙，浏览器可以自如的干原来的事情（显示页面），服务器返回数据的时候通知浏览器一声，浏览器把返回的数据再渲染到页面，局部更新
```

#### 我的理解

```
同步 ：【必须】循序执行 
	具体过程是：客户端提交请求->等待服务器处理->处理完毕返回，在这个过程中客户端（浏览器）不能做其他事。
异步 ： 同时进行的
	具体过程是：客户端发出ajax请求->服务端处理->处理完毕执行客户端回调，在客户端（浏览器）发出请求后，仍然可以做其他的事。
	原理
	js是一门单线程的语言 浏览器只分给js一个主线程 用来执行任务 一次只能执行一个 所以有一个任务队列 浏览器会循序执行主线程中的任务队列 
	异步原理 http请求线程，浏览器定时触发器，浏览器事件触发线程这些都是异步的 浏览器为他们单独开了一个线程  这些异步任务完成后 每个事件都会绑定相应的回调函数 结束时以回调函数的方式返回

	我们肯定不能自己开辟单独的线程，但我们可以利用浏览器给我们开放的这几个窗口，浏览器定时器线程和事件触发线程是好利用的
	记住js是单线程的 他不可能异步执行(语言决定的) 它的浏览器帮它实现的 开辟了新的线程
```
